# AI Review Prompt（极简版）

请对本项目做一次快速代码 review，仅关注高风险问题，并按 `P0/P1/P2` 输出。

必查 4 项：

1. 统计严密性  
- 是否遵循 `.cursorrules`：先做正态性/方差齐性检查，不满足则回退非参数/稳健方法？  
- `p-value` 与效应量计算是否严谨，是否存在多重比较未校正风险？

2. 架构解耦  
- `survey_tools/web/` 是否包含应在 `survey_tools/core/` 的统计逻辑？  
- 是否存在 web/core 双轨重复实现与口径不一致？

3. 异常流与健壮性  
- 全空列、全相同值、异常编码/特殊字符 Excel、无效分组等场景会崩溃还是优雅报错？  
- 是否存在 `except Exception` 吞错导致静默失败？

4. Streamlit 状态管理  
- `st.session_state` 是否有大对象长期驻留、文件切换后旧状态污染、rerun 后状态错乱等风险？

输出格式（严格）：
- 先列问题（按 `P0 > P1 > P2`）；
- 每条包含：`风险` / `证据（文件或函数）` / `影响` / `建议修复`；
- 最后给出：`总评` + `下一步 To Do（最多 5 条）`。
