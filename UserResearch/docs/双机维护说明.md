# 双机维护说明（两台电脑同步迭代本项目）

> 通过 **Git 远程仓库** 在两台电脑之间同步代码与文档，并在两台机器上保持**相同环境与工作方式**，即可实现双机维护与迭代。

---

## 一、前提

- 两台电脑均已安装 **Git**（与可选：Python 3.10+、Streamlit）。
- 有一个可访问的 **远程仓库**（如 GitHub、Gitee、或公司内网 Git 服务器）。

---

## 二、在「第一台电脑」（当前项目所在）上做的准备

### 1. 初始化 Git 并提交当前项目

在**项目根目录**（`问卷数表` 或你当前的仓库根）执行：

```bash
git init
git add .
git status
```

根据 `.gitignore` 会排除虚拟环境、`review_outputs`、大文件等；若你希望把 `input_example` 下示例或部分 xlsx 纳入版本管理，可先调整 `.gitignore` 再 `git add .`。

```bash
git commit -m "初始提交：五剑客 + 规则与验收文档"
```

### 2. 添加远程仓库并推送

将 `YOUR_REMOTE_URL` 换成你的实际地址（如 `https://github.com/你的用户名/问卷数表.git` 或 Gitee 地址）：

```bash
git remote add origin YOUR_REMOTE_URL
git branch -M main
git push -u origin main
```

若远程已有内容（如新建时带了 README），可先 `git pull origin main --rebase` 再 `git push -u origin main`。

---

## 三、在「第二台电脑」上使用项目

### 1. 克隆仓库

```bash
git clone YOUR_REMOTE_URL
cd 问卷数表
```

（若目录名与远程一致，则 `cd` 进入克隆下来的目录。）

### 2. 安装依赖（与第一台保持一致）

推荐使用项目内**同一套依赖文件**，保证两台环境一致：

```bash
cd UserResearch
pip install -r requirements.txt
```

若项目有 `requirements.lock.txt`，可优先用其安装以锁定版本：

```bash
pip install -r requirements.lock.txt
```

### 3. 工作目录与启动方式（与第一台相同）

- **工作目录**：所有脚本从 **`UserResearch`** 运行（或从项目根运行且能解析到 `UserResearch`）。
- **启动工具**：在 `UserResearch` 下执行 `python web_tools_launcher.py` 或 `streamlit run 问卷定量分析工具 v1.py` 等（见 README 与 `AI_Agent_人工验收上下文.md`）。

两台机器上**都用同一套命令、同一工作目录**，避免路径或习惯不一致。

---

## 四、日常双机迭代流程

| 操作 | 电脑 A | 电脑 B |
|------|--------|--------|
| 拉取最新 | `git pull origin main` | 同左 |
| 修改代码/文档 | 改完后 `git add .` → `git commit -m "说明"` → `git push origin main` | — |
| 在另一台继续 | — | `git pull origin main` 后再编辑、提交、推送 |

原则：**先 pull 再改，改完 commit + push**；另一台每次开工前先 **pull**，减少冲突。

若两人或两机同时改同一文件产生冲突，Git 会提示冲突段，按提示合并后再 `git add`、`git commit`、`git push`。

---

## 五、敏感信息与本地独有文件

- **API 密钥、.env**：已通过 `.gitignore` 排除，不要提交。两台机器各自在本地配置 `.env` 或环境变量。
- **大文件**：`review_outputs`、大型 xlsx/pdf 等已忽略，若需在双机共享大文件，请用网盘或共享目录，不要塞进 Git。
- **input_example**：若希望两台都有一份示例数据，可把 `input_example` 下小体积的 csv/xlsx 纳入版本管理（修改 `.gitignore` 中对应规则后 `git add` 再提交）。

## 六、简要检查清单（新机首次）

- [ ] 已安装 Git、Python 3.10+
- [ ] `git clone` 项目并 `cd` 到项目目录
- [ ] 在 `UserResearch` 下 `pip install -r requirements.txt`（或 lock 文件）
- [ ] 配置本地 `.env` 或环境变量（若用 API）
- [ ] 在 `UserResearch` 下运行 `python web_tools_launcher.py` 能启动工具
- [ ] 之后每次开工前执行 `git pull origin main`
