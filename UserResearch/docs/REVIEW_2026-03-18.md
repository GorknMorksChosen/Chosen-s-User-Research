## 一、高风险 Bug（直接导致功能失效或数据严重偏差）

### H1 · 多选组间差异检验永远返回空结果
- **位置**：`survey_tools/web/quant_app.py` ~L1282（`tab_within` 段）
- **原因**：`run_group_difference_test(tmp, seg_col, col, "多选")` 传入的 `col` 是字符串；该函数在多选分支第一步 `isinstance(value_col, list)` 即返回 False，直接返回空结果，P值全 NaN，从未做任何统计检验。
- **修复**：改为 `run_group_difference_test(tmp, seg_col, [col], "多选")` 或改用 `"单选"` 分支（列已预处理成 0/1 二分）。

### H2 · 排序题 Top1/Top2 率分母被 ×n_排序位置
- **位置**：`survey_tools/core/quant.py` L457–477
- **原因**：`tag_counts = long_df["label"].value_counts()` 对转长格式后的行数计数，每位受访者贡献 `len(rank_cols)` 行，分母比真实受访者数大 `n_rank_positions` 倍。
- **修复**：分母改为 `df.groupby(label_col).size()`（受访者人数）。

### H3 · P值/效应量/方向箭头列是完全失效的死代码
- **位置**：`survey_tools/web/quant_app.py` L170（`pivot_v13_style` 函数）
- **原因**：`stats_res = getattr(df_long, "_stats", None)`——整个代码库从未给 DataFrame 设置 `_stats` 属性，该值永远是 `None`，对应显示逻辑（L191–310）从未执行，导出 Excel 的 P值/效应量列永久缺失。
- **修复**：在定量引擎结果（现 `run_quant_cross_engine`，兼容 `run_v13_like_cross`）的每条结果字典中增加 `"stats": run_group_difference_test(...)` 键，`pivot_v13_style` 改从字典键读取。

### H4 · 排序题导出按钮嵌套在父 button 内，点击后立刻消失
- **位置**：`survey_tools/web/quant_app.py` L1421–1514
- **原因**：点击"导出结论表"→ rerun → 父按钮 `if` 块不再渲染 → 导出逻辑未执行 → buffer 未更新 → 下载按钮消失。
- **修复**：将生成结果存 `session_state`，导出/下载按钮移出父 `if` 块。

### H5 · `apply_sav_labels` 新标签与已有列名碰撞，产生重复列
- **位置**：`survey_tools/utils/io.py` L168–180
- **原因**：重命名时只在有标签的列之间去重，不检查新标签是否与无标签的原列名重名，碰撞后 `df["列名"]` 返回 DataFrame 而非 Series。
- **修复**：构建 `rename_map` 前维护 `existing` 列名集合，冲突时继续追加后缀。

### H6 · 传入 `pd.ExcelFile` 时 `read_table_auto` 必然 ValueError
- **位置**：`survey_tools/utils/io.py` L86–102；触发：`satisfaction_app.py` L506–512、`cluster_app.py` L174–181
- **原因**：`pd.ExcelFile` 无 `.name` 属性也非 `str`，三个格式分支全部跳过，直接 `raise ValueError`。多 Sheet Excel 100% 必现报错。
- **修复**：在 `read_table_auto` 入口补充 `isinstance(path_or_file, pd.ExcelFile)` 分支。

### H7 · 0/1 编码多选列被误判为"评分"
- **位置**：`survey_tools/core/question_type.py` L18–34
- **原因**：`detect_column_type` 先做 numeric/评分检测（L23–31），0/1 编码列满足"≤11 unique，范围 [0,1]"被归为"评分"；多选 prefix 检查（L32）在其后永远不生效。
- **修复**：将 `prefix in multi_prefixes` 检查提前到 numeric 前；补充 `set(uniq).issubset({0.0, 1.0})` 时归类"多选"。

### H8 · IPA 四象限标签在负相关时坐标倒置
- **位置**：`survey_tools/web/satisfaction_app.py` L94–98
- **原因**：标签坐标 `y_mean * 1.1 / * 0.9`，当 `y_mean < 0` 时方向反转，"重点保持区"与"过度服务区"互换，严重误导优化方向判断。
- **修复**：改用轴范围四分位点定位而非均值比例缩放。

### H9 · `df_clean` 与 `df_for_clustering` 行数不一致无任何校验
- **位置**：`survey_tools/web/cluster_app.py` L274–291
- **原因**：`perform_factor_analysis` 内部若有额外 `dropna()`，输出行数可能少于 `df_clean`，后续 `perform_clustering` index 对齐失败，分群结果错误且无提示。
- **修复**：在 L291 后添加显式行数断言并给 UI 提示。
