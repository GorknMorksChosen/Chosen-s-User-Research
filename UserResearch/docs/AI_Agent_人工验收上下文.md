# AI Agent 人工验收上下文（供智能体快速接手）

> 本文档为搭建 AI agent（如 Open Claw 类）完成**16 条映射验收**而整理，便于智能体在对话重开或新会话中快速理解前情与任务，无需回溯完整对话。

---

## 一、项目与任务一句话

- **项目**：问卷数表分析工具集，含 5 个核心 Web 工具（五剑客），用于问卷数据的交叉分析、满意度建模、归因/分群、文本分析等。
- **你的任务**：按《映射验收表》逐条**人工验收**（表内共 18 条；每条 = 运行指定工具 + 上传指定数据 + 在界面/导出中确认检查点），将每条标为「通过」或「不通过」；不通过项写入《验收问题清单》。

---

## 二、工作目录与入口（统一从 UserResearch 运行）

所有命令的**工作目录**为项目下的 **`UserResearch`**（即 `问卷数表/UserResearch` 或你当前仓库中的等价路径）。

| 工具简称 | 入口脚本（相对 UserResearch） | 启动方式示例 |
|----------|-------------------------------|------------------------|
| Quant | `问卷定量分析工具 v1.py` | `streamlit run 问卷定量分析工具\ v1.py --server.port 8501` |
| Standard | `超级应用_满意度与体验建模.py` | `streamlit run 超级应用_满意度与体验建模.py --server.port 8502` |
| 游研专家 | `game_analyst.py` | `streamlit run game_analyst.py --server.port 8503` |
| 聚类 | `聚类.py` | `streamlit run 聚类.py --server.port 8504` |
| Text | `问卷文本分析工具 v1.py` | `streamlit run 问卷文本分析工具\ v1.py --server.port 8505` |

也可用统一启动菜单（支持多选）：在工作目录执行 `python web_tools_launcher.py`，按提示输入如 `1,2,3` 启动多个工具。

## 四、16 条验收一览（条目 → 用哪个工具 → 检查什么）

智能体需能：**启动对应 Streamlit 应用 → 上传 input_example 数据 → 按「检查点」在界面或导出中确认**，然后将该条在《映射验收表》中标为「通过」或「不通过」。

| # | 清单条目 | 使用工具 | 检查点（一句话） |
|---|----------|----------|------------------|
| 1 | 探索数据特征（描述统计、分布） | Quant / Standard / 游研 / 聚类 | Quant：选评分题+分组，有描述或交叉汇总。Standard：有 IPA 均值/相关。游研/聚类：有数据体检或缺失/特征预警。 |
| 2 | 定类数据探索分析 | Quant | 单选题做交叉表，有题型识别、频数或定类分布。 |
| 3 | 定量数据探索分析 | Quant、聚类 | Quant：评分/矩阵题有均值、标准差或分组结果。聚类：有缺失预警、特征列表或聚类前描述。 |
| 4 | 数据分布之正态性分析（检验法） | Quant | 定量题+分组做分组定量；界面或导出中可见 ANOVA 前有正态/方差齐性检查或选用 Kruskal 的说明。 |
| 5 | 异常值处理 | 游研专家、聚类 | 游研：数据体检报告含直线勾选、作答过快、离群值。聚类：切换缺失值策略（剔除/均值/中位数等）可跑通且结果随策略变化。 |
| 6 | 标题处理及其他 | Quant | 带【单选】题干的数据；无关列（如 Type_）被屏蔽，题干用于题型识别，选错题有跳过或提示。 |
| 7 | t 检验（独立样本多组→ANOVA/Kruskal） | Quant | 定量题+分组（多组）；输出为 ANOVA 或 Kruskal-Wallis，含事后比较（Tukey 或 Dunn）。 |
| 8 | 方差分析（单因素 + 事后比较） | Quant | 同上；有 p 值、效应量、Tukey HSD 或 Dunn 两两比较。 |
| 9 | 卡方检验（2×2、R×C、Fisher） | Quant | 两道单选题交叉表；有列联表、卡方 p 值；2×2 小期望时见 Fisher。 |
| 10 | 非参数（Kruskal-Wallis + Dunn） | Quant | 定量+多组；不满足正态/方差齐性时见 Kruskal-Wallis 与 Dunn 事后。 |
| 11 | 相关分析（IPA 用相关系数） | Standard | 满意度应用上传细项+整体满意度数据；IPA 四分图「重要性」为相关系数，可区分四象限。 |
| 12 | 线性回归（多重、VIF、信度） | Standard、游研 | Standard：回归输出含系数、p 值、R²、VIF、Cronbach α。游研：有回归或驱动力输出。 |
| 13 | 单选题与多选题分析（频数、卡方） | Quant | 单选×分组、多选×分组交叉；有频数/占比、卡方或 Fisher、显著性标识（箭头/色阶）。 |
| 14 | 信度分析（Cronbach α） | Standard、游研 | Standard：回归/信度模块展示 Cronbach's α。游研：量表分析有信度系数。 |
| 15 | 路径分析 | 游研专家 | 路径分析步骤指定变量与路径；有路径估计结果（如 semopy 系数或拟合信息）。 |
| 16 | 结构方程模型分析 | 游研专家 | 同上；SEM 可配置并输出估计结果。 |
| 17 | 聚类分析（多算法、因子降维等） | 聚类、游研 | 聚类：可选 KMeans/GMM/Agglomerative、因子降维、手肘/轮廓图、K 与算法推荐、结果与画像。游研：分群有 K-Means 结果。 |
| 18 | 文本分析（开放题语义/编码/摘要） | Text | 上传含开放题表格；可做语义分析、关键词或情感、编码与摘要，并能导出（多 sheet 或下载）。 |

*映射验收表中为 18 个验收项（路径分析与结构方程各一条）；汇总处若写 16 系历史口径，以实际表内条数为准。*

## 五、智能体需完成的动作

1. **按上表或《映射验收表》顺序**，对每条执行：启动对应工具 → 上传 `input_example` 下数据 → 按检查点查看界面或导出。
2. **更新《映射验收表》**：将该条的「验收状态」从「待验收」改为「通过」或「不通过」；全部完成后更新文末「验收状态汇总」的通过/不通过数量（表内共 18 条，汇总数量与之一致）。
3. **若某条不通过**：在《验收问题清单》「二、16 条映射验收」中**新增一行**：清单条目、对应五剑客、**现象**（界面报错/导出缺项/结果不符）、**可能原因**（可选）。

- 项目以《SPSSAU 问卷/量表数据分析方法与应用》为参考整理了「功能可执行清单」，并将清单六大类与五剑客做了对应（已实现/部分实现/未实现），见 `UserResearch/五剑客_wjxspss目录_对应表.md`。

## 八、智能体能力假设

- 能**启动 Streamlit 应用**（指定脚本与端口），并在**浏览器或无头环境中打开界面**。
- 能**上传本地文件**（如 `input_example/腾讯问卷导出_MMORPG游戏调研.csv`）到 Streamlit 文件上传组件。
- 能**按步骤操作界面**（选择题目、分组、点击分析/导出等），并**读取界面文本或导出文件**以判断检查点是否满足。
- 能**编辑 Markdown 文档**：修改《映射验收表》中的验收状态与汇总、在《验收问题清单》中追加不通过行。

若智能体无法直接操作浏览器，可改为：**输出逐条「操作步骤 + 检查点」清单**，由人类按步骤执行并在文档中勾选；或由人类执行后把结果（通过/不通过及现象）提供给智能体，由智能体更新上述两文档。
