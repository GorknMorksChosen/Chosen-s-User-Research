## P1（提升覆盖率与一致性，适合 2～4 周推进）

- **问卷星表头自动规范化**
  - **目标**：消除「多选/矩阵题仅第一列带 Qn 题干、其余列为纯选项」导致的无法分组问题；工具内加载时自动检测并规范化表头，无需用户手动改表头。
  - **实现**：见 `UserResearch/设计_问卷星表头规范化.md`。`survey_tools.utils.wjx_header.normalize_wjx_headers` 在读入后、题型解析前统一调用；可选独立脚本 `UserResearch/scripts/normalize_wjx_headers.py` 生成「表头修改后」文件。

- **.sav 变量/值标签可选应用**
  - **目标**：读 .sav 时用 `load_sav` 获取变量标签与值标签，在界面中可选将列名替换为题干、将编码替换为选项文字，与 CSV/Excel 体验一致。
  - **实现方案**：见 `UserResearch/设计_sav变量与值标签可选应用.md`（io 层新增 `read_table_auto_with_meta`、`apply_sav_labels`；各工具入口 .sav 时提供「应用变量标签」「应用值标签」勾选；默认不应用，兼容现有行为）。
  - **验收**：上传带标签的 .sav，勾选应用后交叉表/导出为题干与选项文字；CSV/Excel 行为不变。

- **入口与 core 双轨收口（重复实现治理）**
  - **现状**：`game_analyst.py` 与 `survey_tools/core/advanced_modeling.py` 存在重复 Analyzer（报告问题 10）。
  - **建议落点**：UI 入口只保留展示与参数收集；计算统一调用 core。

- **相关分析补齐：偏相关（Standard）**
  - **来源**：wjxspss 清单“相关分析：两变量相关、偏相关”；当前偏相关未实现。
  - **建议落点**：core 增加偏相关计算（可用 pingouin/partial_corr 或等价实现），web 增加入口。

- **差异检验补齐：配对/单样本 t 与配对/单样本非参数（Quant）**
  - **来源**：wjxspss 清单；当前主要覆盖多组 ANOVA/Kruskal。
  - **建议落点**：core/quant 增加 API + 在 Quant UI 增加“同一样本前后”类分析模式。

- **聚类导出升级：从 CSV 提升为 xlsx 多 sheet（聚类）**
  - **建议落点**：复用 `ExportBundle/export_xlsx`，sheet 拆分为“评估指标/推荐结果/画像/样本标签”。

- **多选事后检验与显著性矩阵（Quant）**
  - **目标**：与 v1.3 对齐，对校正后显著的多选选项做组间两两 Fisher 事后检验并 FDR 校正，生成组×组显著性矩阵或 posthoc_arrows，供 pivot/导出展示。
  - **参考**：`UserResearch/问卷数表分析工具 v1.3.py` 约 1698–1888 行；方案见 `v1.3_多选矩阵经验迁移至五剑客` 计划。

- **题型标记按题号视图（Quant）**
  - **目标**：在题型微调区增加「按题号查看」视图或折叠面板：按题号列出题干摘要与当前题型，支持整题一键设为多选/矩阵/忽略，减少多选/矩阵题多列逐列修改。
  - **参考**：v1.3 的 `TypeMarkerDialog`（按题号 + 题干 + 题型下拉）。

- **矩阵题命名/导出一致化（可选）**
  - **目标**：导出时对矩阵题列名做显示用别名（如 `矩阵单_Q_子项`、`矩阵评分_Q_子项`），与 v1.3 命名风格一致；分析管线可保持现有 `Qn. stem: 子项`。

- **用户标签列识别与手动列归属（Quant）**
  - **问题**：表头中「type.玩家分类 A1」等用户标签列被表头规范化或分组逻辑误归入前一题的选项，且无法在工具内将其单独视为一题或设为忽略。
  - **目标**：① 正确识别：列名以 `type.`（不区分大小写）开头的列视为用户标签，不在问卷星表头规范化时归入前一题，默认题型为「忽略」。② 可选：支持在题型微调中「将选中列单独设为一道题」或「从某题中拆出为忽略/独立列」，便于把被错误归入某题的列捞出来单独处理。
