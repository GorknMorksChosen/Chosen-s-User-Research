# 问卷星表头自动规范化

## 问题

- 定量工具（尤其 Quant）依赖「每题下每列均为“Qn. 题干: 选项”」才能正确把多选/矩阵题的多列归到同一题。
- 问卷星原始导出：第 1 列为「Qn. 题干+首选项」（无冒号），后续列为纯选项名（无 Q 前缀），导致 `extract_qnum` 仅能识别第一列，其余列被忽略。

## 方案

1. **工具内自动规范化（推荐）**  
   在读入数据后、交给题型解析前，对列名做一次「问卷星原始 → 修改后」的规范化，下游逻辑无需改动。  
   - 模块：`survey_tools.utils.wjx_header.normalize_wjx_headers(df) -> (df, bool)`  
   - 接入：Quant、Standard（满意度）、聚类、游戏体验、问卷文本分析 五剑客在「得到 df」处统一调用；若返回 `True` 可提示「已自动规范化问卷星表头」。

2. **独立预处理脚本（可选）**  
   - 脚本：`UserResearch/scripts/normalize_wjx_headers.py`  
   - 用法：`python scripts/normalize_wjx_headers.py 输入.xlsx [-o 输出.xlsx] [-s 0]`  
   - 用途：先得到「表头修改后」文件再在 Excel 中编辑或给他人使用。

## 行为要点

- **检测**：存在「以 Q数字 开头」的列且其后有「不以 Q数字 开头」的列时，视为问卷星原始格式。
- **列名重写**：题干+首选项列统一为「Qn. 题干: 选项1」；纯选项列改为「Qn. 题干: 选项文字」；题干优先按 `]` 后截断，其次按 `？`/`?`。
- **兼容**：已是「Qn. 题干: 选项」或单题单列时不改写（返回 `(df, False)`）；首列「Q序号」「序号」等不参与分组。

## 验收

- 单元：问卷星原始风格小 DataFrame 经 `normalize_wjx_headers` 后，多选/矩阵题各列均为「Qn. 题干: 选项」且题号一致；已规范化表头则 `was_modified is False`。
- 回归：Quant 用原始 xlsx 上传不手动改表头，多选/矩阵题能正确分组并参与交叉分析。
